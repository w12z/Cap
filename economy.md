# Cap 核心经济模型设计

> 状态：讨论稿 v0.1
>
> 本文档是后续经济系统实现的唯一设计基线。当前阶段只讨论和记录模型，不编写代码。
> 已确认内容标记为“确定”，尚需讨论的内容标记为“提案”或“待定”。

## 1. 设计目标

### 1.1 确定原则

- 游戏首先是一个可运行、可观察、可干预的经济模拟，其次才是公司管理游戏。
- 国家、公司、居民和金融机构之间必须双向影响，不能只给玩家公司添加固定增益或减益。
- 经济结果应由资产负债、收入支出、生产、库存、信用和预期共同产生，而非通过预设剧情强行指定。
- 同一种冲击在不同制度、产业结构和资产负债表下应产生不同结果。
- 模型需要足够严谨，但不逐个模拟每位居民，也不追求学术模型的全部细节。
- 玩家应能理解主要因果链，例如“加息 → 信贷收缩 → 投资下降 → 失业上升 → 需求下降”，不能只看到随机数字变化。

### 1.2 严谨性的定义

本游戏中的“严谨”主要指：

1. **存量与流量一致**：存款、贷款、债券、股权、现金、库存和资本设备都不能凭空出现或消失。
2. **复式记账一致**：一方的金融资产必须是另一方的金融负债；每笔交易至少影响两个账户。
3. **国民经济核算一致**：GDP 的生产法、收入法和支出法应在允许的统计误差内一致。
4. **市场过程可解释**：价格不是随机波动，而是由订单、库存、产能、成本、竞争和预期推动。
5. **制度通过规则生效**：资本主义、社会主义或混合经济不是简单标签，而是一组所有制、价格、信贷、财政和贸易规则。

## 2. 模拟尺度

### 2.1 经济主体采用分层聚合

不逐人模拟全国人口，而是使用“代表性群体 + 关键独立机构”的混合模式。

- 居民按收入、职业、财富和消费偏好聚合为若干群体。
- 中小企业可以按行业聚合为竞争者群体。
- 玩家公司、大型企业、主要银行、央行、政府和大型保险公司单独模拟。
- 国家之间同时运行宏观模型，并通过贸易、资本流动和汇率连接。

### 2.2 时间节拍提案

- **每日**：商品订单、零售销售、证券交易、支付结算、汇率变化。
- **每月**：生产、工资、租金、利息、保费、贷款偿还、库存盘点、价格调整。
- **每季度**：企业财报、银行风险评估、GDP 核算、政策会议、投资计划调整。
- **每年**：人口和劳动力变化、长期资本折旧、税率与财政预算、产业结构调整。

玩家可暂停，并在 1×、2×、4× 等速度间切换。经济结算按固定顺序执行，避免系统结果取决于画面帧率。

## 3. 核心会计框架

### 3.1 主要账户

每个经济主体至少包含：

- 现金与央行货币
- 银行存款
- 应收账款与应付账款
- 贷款
- 债券资产与债券负债
- 股权资产与所有者权益
- 商品库存
- 固定资本和累计折旧
- 当期收入、成本、税费与利润

并非所有主体都拥有全部账户。例如居民通常没有商品库存，央行通常没有生产性固定资本。

### 3.2 基本约束

资产负债表恒等式：

```text
资产 = 负债 + 所有者权益
```

支出法 GDP：

```text
GDP = 居民消费 C + 投资 I + 政府购买 G + 出口 X - 进口 M
```

收入法 GDP：

```text
GDP = 工资 + 利润 + 利息净收入 + 租金 + 生产税净额 + 折旧
```

部门余额必须闭合。一个部门的金融盈余必然对应其他部门的金融赤字。

### 3.3 货币创造原则

- 商业银行发放贷款时，同时创造贷款资产和客户存款负债。
- 贷款本金被偿还时，相应存款货币被销毁。
- 银行之间通过央行准备金结算。
- 央行通过政策利率、准备金、公开市场操作和流动性工具影响信用条件。
- 央行不是所有货币的唯一直接创造者，商业银行信用创造是货币系统的重要部分。

## 4. 经济主体

### 4.1 居民部门

居民采用动态聚合群体，不逐户模拟。初版建立约 12 至 20 个全国居民原型，例如：

- 低收入劳动者
- 中等收入劳动者
- 高技能与高收入劳动者
- 小业主与个体经营者
- 资本所有者
- 失业及福利领取者
- 退休者

居民原型不是固定阶级标签，而是由就业状态、技能、主要收入来源、收入水平、财富负债状态和消费偏好共同定义。人口可以在原型之间流动。

为了表现公司的局部影响，每个实际居民群体单元由“居民原型 + 居住地区”组成。例如同属中等收入技术劳动者，居住在不同地区时拥有不同的通勤选择、房租、就业机会和生活成本。

每个群体具有：

- 人口与劳动参与率
- 就业人数、失业人数和工资水平
- 可支配收入
- 存款、贷款、债券和股票财富
- 必需品与可选消费品偏好
- 边际消费倾向和储蓄倾向
- 对价格、收入、利率和失业风险的敏感度
- 信心与通胀预期
- 居住地区、可接受通勤范围和迁移倾向

居民消费预算由可支配收入、财富、消费信贷和预期共同决定。必需品优先满足，可选消费品之间允许有限替代。

居民群体可因就业、失业、晋升、创业、退休、财富积累、债务恶化和迁移而发生人口转移。转移具有时间延迟，不会在一个结算期内瞬间完成。

### 4.2 企业部门

企业分为玩家公司、独立大型企业和行业聚合竞争者。

每个企业至少具有：

- 所有制：私人、国有、合作社、混合所有制
- 所属行业与生产技术
- 厂房、设备、产能和资本折旧
- 雇员、工资与劳动生产率
- 原材料、在制品和成品库存
- 供应合同、销售订单和交付周期
- 现金、存款、贷款、债券和股权
- 成本、定价、投资、融资与分红策略
- 信用等级、违约概率和破产状态

初期玩家公司的微观管理可以简化，但资产负债、生产投入、库存、销售和融资必须进入统一经济系统。

### 4.3 政府与财政部门

政府具有：

- 税收：个人所得税、企业所得税、消费税、关税、财产税等
- 支出：公共消费、政府采购、转移支付、补贴、公共投资
- 国债余额、期限结构和利息支出
- 财政账户、赤字、债务率和信用水平
- 产业政策、价格干预、国有资产和社会保障制度

政府购买计入 GDP；转移支付不直接计入 GDP，但会改变居民可支配收入与消费。

### 4.4 央行

央行具有：

- 政策利率与利率走廊
- 商业银行准备金账户
- 政府债券、外汇储备和对银行贷款
- 通胀目标、就业目标或汇率目标
- 法定准备金率、流动性工具与宏观审慎工具
- 可信度、独立性和政策反应函数

央行政策不直接决定市场结果，而是通过融资成本、信贷供给、汇率、资产估值和预期传导。

### 4.5 商业银行

银行具有：

- 居民和企业存款
- 企业贷款、住房贷款、消费贷款和政府债券
- 央行准备金与同业资产负债
- 资本、利润、坏账准备和不良贷款
- 资本充足率、流动性覆盖率和风险偏好
- 存款利率、贷款利率和授信标准

银行不能只看准备金决定是否放贷；贷款需求、借款人偿付能力、银行资本和风险偏好共同决定信贷供给。

### 4.6 保险公司

保险公司具有：

- 保单数量、保费收入和承保风险
- 未到期责任准备金与赔付准备金
- 债券、存款和股票投资组合
- 偿付能力资本与再保险安排
- 索赔发生率、灾害冲击和经营费用

保险的核心作用是重新分配风险，而不是消除总体损失。重大灾害可能同时冲击企业、银行、保险和政府财政。

### 4.7 证券市场参与者

- 上市企业
- 居民投资者群体
- 银行和保险机构
- 基金及其他机构投资者
- 政府与央行
- 外国投资者

证券包括股票、公司债和政府债。衍生品暂不进入初版核心范围。

### 4.8 外国部门

外国部门记录：

- 商品和服务进出口
- 外债与外国资产
- 跨境直接投资和证券投资
- 汇率、资本管制和外汇储备变化
- 贸易条件、全球商品价格和外部冲击

## 5. 国家与经济制度

### 5.1 不使用单一制度标签决定全部行为

“资本主义国家”“社会主义国家”等标签只用于向玩家解释整体倾向。实际模拟由以下制度参数组成：

- 私人所有权保护程度
- 国有经济占比
- 行业准入与私人资本限制
- 价格自由度和价格管制范围
- 市场竞争强度与反垄断力度
- 劳资议价能力和最低工资
- 税负与福利水平
- 财政支出结构
- 央行独立性和政策目标
- 银行国有化程度与信贷指导强度
- 资本账户开放程度
- 贸易开放程度和关税
- 证券市场开放度
- 法治、腐败和行政效率
- 破产保护与债权人保护
- 数据透明度和政策可预期性

### 5.2 制度原型提案

初版国家可以从以下原型组合产生，但每个国家仍拥有独立参数：

1. **自由市场型发达经济体**：消费和金融发达、创新强、竞争激烈、工资和经营成本高、金融周期明显。
2. **社会市场型发达经济体**：福利和劳动保护强、内需稳定、税负较高、政策可预期。
3. **国家主导型工业经济体**：国有银行与产业政策强、基础设施投资高、重点行业准入受限、信贷配置可能扭曲。
4. **计划协调型社会主义经济体**：国有企业占比较高、价格和信贷受到指导、就业稳定但供需信号可能失真。
5. **新兴市场经济体**：增长和人口红利较高、资本与外汇约束明显、金融体系脆弱、政策波动较大。
6. **资源出口型经济体**：财政和汇率依赖资源价格、进口消费占比高、荷兰病风险明显。

不存在绝对强弱国家，只有不同机会、约束和风险结构。

## 6. 商品系统与生产

### 6.1 商品边界（已确认）

商品是卖方可以交付，买方在交易后可以取得一个可识别产品、数量或使用权的经济对象。

- 商品可以是实体产品，例如粮食、钢材、设备、汽车和药品。
- 商品也可以是数字产品，例如电子游戏、标准软件、电子书和数字内容许可。
- 数字商品不要求转移实体，但买方必须获得可识别的副本、账户内容或许可权。
- 电力等难以库存的产品仍可属于商品，因为买方取得了可计量、可交付的能源单位。
- 商品可以作为中间投入、资本品或最终消费品。
- 商品不包含运输、物流、医疗、教育、广告、定制研发等履约活动。
- 股票、债券、贷款和保险合同不是商品，而是金融工具或金融合同。

商品与服务的边界不以“有形/无形”判断，而以交易对象判断：

```text
买方取得可识别产品、数量或许可权 → 商品
买方购买一定时间、能力或结果的履约 → 服务
买方取得未来现金流、偿付或风险保障权利 → 金融工具/金融合同
```

### 6.2 商品属性

每类商品根据自身性质具有不同属性：

- 可储存、不可储存或具有有限保质期
- 实体商品或数字商品
- 消耗品、耐用品或资本品
- 标准化产品或具有品牌、质量差异的产品
- 是否需要运输、仓储和零售服务才能送达买方
- 是否形成居民耐用品存量或企业固定资本
- 复制成本、研发成本和知识产权保护程度

数字商品应允许“前期研发成本高、复制边际成本低”的商业模式，不能简单套用实体商品的库存与产能规则。

### 6.3 分层商品目录（已确认）

初版不从 50 类以上的细商品开始，但商品目录必须天然支持上下级关系和后续拆分。

商品目录采用树状分类：

```text
食品
├─ 农业原料
│  ├─ 粮食
│  └─ 油料作物
├─ 动物产品
│  ├─ 肉类
│  └─ 乳制品
└─ 加工食品

电子产品
├─ 半导体
│  ├─ 晶体管/基础器件
│  └─ 集成电路
├─ 电子元件
└─ 计算机
```

分类树只表达“属于什么类别”，不直接表达生产投入。例如肉类与粮食都属于食品体系，但肉类不是粮食的分类子项。

目录节点分为：

- **统计节点**：用于汇总价格、产量、消费和产业规模，本身通常不可直接交易。
- **可交易叶子**：当前模拟精度下实际下单、库存、定价和交付的商品。
- **折叠节点**：初期可作为粗粒度商品交易，后期拆分为多个更具体的叶子商品。

当一个折叠节点被细分时，旧库存、历史价格、生产配方和合同必须按照明确比例迁移，不能破坏既有会计记录。

### 6.4 生产依赖图（已确认）

商品的上下级分类与生产依赖是两个独立结构。生产关系使用有方向的配方图表示：

```text
粮食 + 水 + 养殖设备 + 劳动
  → 养殖过程
  → 肉类

硅材料 + 金属材料 + 化工材料 + 半导体设备 + 技术劳动
  → 晶体管/集成电路
  → 电子元件
  → 计算机
```

每个生产配方至少描述：

- 必需商品投入及单位投入系数
- 可替代商品组及替代效率，例如不同饲料来源
- 所需劳动技能与工时
- 所需资本设备和有效产能
- 所需能源商品
- 所需运输、维修或专业服务容量
- 生产周期、损耗率和良品率
- 主产品、副产品和废弃物

服务可以成为生产配方的必要履约条件，但只消耗服务容量或完成合同，不进入商品库存。例如计算机制造需要物流服务交付芯片，但“物流”不会成为计算机库存中的一种商品。

生产投入的替代规则已经确认：

- **关键投入**不可替代，缺失时对应生产阶段停止。例如计算机不能在完全缺少半导体元件时正常生产。
- **类别投入**可以接受指定分类下的多个子商品。例如粗粒度配方可要求“电子元件”，细分后再限定具体元件组合。
- **替代投入组**只允许技术上可行的替代品，并分别设置换算效率、成本、质量和设备兼容影响。
- 替代不会仅因某种商品价格便宜而自动发生；企业必须具备对应技术、设备或供应认证。
- 关键投入和可替代投入可以同时存在。例如肉类生产必须有有效饲料投入，但饲料可由粮食、豆粕等按不同效率组合。

生产图初期以无循环产业链为主。回收、再制造等循环链条后续可以加入，但必须具有时间延迟和损耗，防止无成本循环生产。

### 6.5 生产函数提案

初版采用带瓶颈的固定投入系数模型，辅以产能利用率和生产率：

```text
可生产数量 = min(
  各类原材料库存 / 单位投入系数,
  有效劳动 / 单位劳动需求,
  有效资本产能
)
```

生产率受技术、设备质量、员工技能、管理水平、基础设施和产能利用率影响。

这样可以形成真实的供应链瓶颈，而不会出现缺少关键零件仍可正常生产的情况。

### 6.6 库存与产能

- 企业根据预期销售、当前订单和目标库存安排生产。
- 库存不足会导致缺货、延迟交付和涨价压力。
- 库存过高会导致促销、减产、现金流紧张和坏账风险。
- 新增产能需要资本品、建设时间和融资，而不是立即生效。
- 固定资本逐期折旧，维护不足会降低有效产能。

## 7. 服务系统

### 7.1 服务边界（已确认）

服务交易的对象是一定时间、能力、过程或结果的履约，而不是可独立转售的产品库存。

- 运输和物流服务改变商品的位置和交付状态。
- 仓储服务提供一定时期的储存能力，但不取得客户商品所有权。
- 建筑服务消耗劳动和商品，最终形成建筑或基础设施资产。
- 维修服务恢复设备或耐用品的有效能力。
- 医疗、教育、广告、咨询和定制研发以履约过程或结果结算。
- 零售与批发提供渠道、展示、撮合和交付组织能力，不与其销售的商品混为一类。

服务通常不能像实体商品一样形成成品库存。未使用的服务能力表现为空闲运力、空置工时或闲置设施，而不是库存积压。

### 7.2 服务容量与合同

服务企业至少具有：

- 可用容量，例如车辆运力、仓库面积、床位、工程团队工时
- 服务区域、交付周期和排队长度
- 员工技能、设备和必要商品投入
- 服务质量、可靠性和违约率
- 即时订单、长期合同和容量预订
- 固定成本、单位履约成本和容量利用率

服务供不应求时表现为排队、延迟、拒单和涨价；需求不足时表现为空闲容量、降价、裁员和设施退出。

### 7.3 商品与服务的组合交易

一笔消费可以同时包含商品和服务，但账户必须分开记录。例如：

- 购买电子游戏本体属于数字商品；订阅服务器、云存档或在线会员属于服务。
- 购买汽车属于商品；维修、保险和运输分别属于维修服务、金融合同和运输服务。
- 建筑材料属于商品；施工属于服务；竣工建筑成为固定资本或房地产资产。
- 标准软件许可属于数字商品；定制开发、部署和维护属于服务。

### 7.4 金融活动不并入通用服务市场

银行、保险和证券市场拥有独立系统。银行手续费、资产管理费等可在 GDP 核算中形成金融服务增加值，但贷款、存款、股票、债券和保险责任本身仍按金融资产、负债和合同处理，不作为普通商品或普通服务订单流转。

## 8. 商品市场与价格

### 8.1 分散报价与非瞬时出清（已确认）

现实中的多数商品市场不是每个周期都通过统一拍卖形成均衡价格。初版建议采用“企业报价 + 消费者选择 + 库存反馈”的非瞬时出清机制。

企业定价考虑：

- 单位成本和目标利润率
- 库存偏离目标的程度
- 未完成订单和交付周期
- 产能利用率
- 竞争者价格
- 预期通胀与融资成本
- 市场份额目标

消费者和企业买方考虑：

- 价格
- 产品质量
- 品牌与可靠性
- 交货时间
- 既有供应关系
- 替代品可得性

同一种商品可以由不同企业以不同价格销售。市场公布的参考价格是实际成交价格的加权统计值，不是所有企业必须采用的统一价格。

### 8.2 消费者、产品与品牌选择

消费者不逐个模拟。每个地区居民群体根据自身收入、偏好和既有经验，在可接触的产品之间分配购买份额。

公司、品牌与产品是三个不同层次：

- 公司负责资产、生产、融资与总体经营。
- 品牌积累认知、口碑、定位和忠诚度，一家公司可以经营多个品牌。
- 产品具有具体价格、规格、质量、故障率、包装、保修和库存。

产品竞争力不能压缩为一个固定“质量分数”，而是使用与商品类别相关的质量维度。例如：

- 食品：安全、口味、营养、新鲜度
- 汽车：性能、可靠性、安全、能耗、舒适度
- 计算机：性能、稳定性、耐用性、兼容性
- 电子游戏：内容质量、技术稳定性、完成度、更新支持

居民群体对各维度具有不同权重。低收入群体通常更重视支付压力和耐用性，高收入群体可能更重视性能、体验、品牌和售后，但这些倾向由国家与群体数据决定，不写成绝对规则。

### 8.3 真实质量、认知与口碑

以下变量必须分开：

- **真实质量**：由设计、技术、投入品质量、设备、员工技能、生产管理和质量控制形成。
- **消费者认知**：消费者在购买前对质量的估计，受规格信息、广告、媒体、价格信号和品牌历史影响。
- **使用体验**：购买后实际获得的效用、故障、退货和耐用表现。
- **品牌口碑**：过去消费者体验经评价、熟人传播、媒体和事件逐步累积的结果。

高质量新品牌不会立刻获得全国信任；高广告投入可以提高知名度和短期认知，但长期无法完全掩盖高故障率或虚假宣传。

口碑更新具有滞后和不对称性：普通正面体验缓慢积累信誉，严重质量事故、召回、拒绝保修或欺诈会造成更快、更大的负面影响。

口碑应按商品类别分别积累。消费者可能信任某公司的计算机品牌，但不因此自动信任它新推出的食品品牌。

### 8.4 售后服务

售后承诺属于产品报价的一部分，但维修、退换、客服和更新支持由独立服务系统履约。

售后评价至少考虑：

- 保修期限和覆盖范围
- 维修网点或在线支持的可达性
- 维修人员、备件和服务器容量
- 平均响应与完成时间
- 拒保率、退换货执行和争议处理
- 企业持续提供更新与备件的能力

承诺本身不能直接产生高售后评分。企业必须支付人员、设施、备件和服务合同成本；容量不足会形成排队、延迟和口碑损失。

### 8.5 群体选择过程

每个居民群体按以下顺序形成需求：

1. 根据收入、现有耐用品、价格和预期判断是否产生购买需求。
2. 依据品牌知名度、零售覆盖、地区可得性和法规形成候选产品集合。
3. 计算每个候选产品对该群体的综合吸引力。
4. 在候选产品和“不购买/延期购买”之间分配需求份额。
5. 根据库存、交付能力、消费信贷和服务容量完成交易或形成缺货。
6. 购买后的使用体验更新退货、口碑、忠诚度和下一期需求。

综合吸引力提案：

```text
吸引力 = 各质量维度 × 群体偏好权重
       + 品牌口碑与既有忠诚
       + 售后和可得性
       - 相对收入的价格负担
       - 交付等待、预期维修和转换成本
```

群体内部仍存在偏好差异，因此不会让得分最高的产品自动获得全部消费者。模型使用平滑份额分配，使领先产品获得更高概率，同时允许细分品牌和低价品牌共存。

耐用品还需要考虑更换周期、旧产品存量、维修选择和二手市场（后期）；数字商品则需要考虑设备兼容、网络效应、持续更新和账号生态锁定。

### 8.6 供需不平衡的表现

供给不足不会被强制“数学清零”，而会表现为：

- 缺货和积压订单
- 交付时间延长
- 价格上涨
- 进口增加
- 企业扩产和新企业进入

需求不足会表现为：

- 库存累积
- 降价和促销
- 减产、裁员和投资下降
- 企业退出或破产

### 8.7 价格指数

- 居民消费价格指数 CPI
- 生产者价格指数 PPI
- GDP 平减指数
- 房地产或资产价格指数（后期）

指数使用实际交易数量作为权重，并定期更新消费篮子，避免仅由报价变化制造虚假通胀。

## 9. 劳动力市场

居民群体提供不同技能等级的劳动。企业发布岗位、工资和技能要求。

工资变化考虑：

- 行业劳动力供需
- 失业率
- 劳动生产率
- 通胀预期
- 最低工资和集体谈判
- 员工流动成本
- 企业盈利和支付能力

失业不会立即消失。技能错配、地域限制和招聘周期会形成摩擦性与结构性失业。

## 10. 信贷与银行体系

### 10.1 企业贷款审批

银行评估：

- 经营现金流与利息覆盖率
- 资产负债率
- 抵押品价值
- 行业风险和宏观环境
- 企业历史违约记录
- 贷款期限与银行自身资本约束

贷款利率建议由以下部分构成：

```text
贷款利率 = 政策利率/资金成本 + 期限溢价 + 信用风险溢价 + 银行利润率
```

### 10.2 银行危机传导

企业亏损 → 贷款逾期 → 银行坏账与资本下降 → 授信收紧 → 企业融资困难 → 投资与就业下降。

政府可选择存款保险、流动性救助、资本重组、接管或允许银行破产，不同政策会产生财政成本、道德风险和信心影响。

## 11. 央行与货币政策

### 11.1 政策反应提案

在通胀目标制国家，央行可参考类似泰勒规则的政策框架，但保留制度差异和人为偏好：

```text
目标政策利率 = 中性利率
             + 通胀偏离目标的反应
             + 产出/就业缺口的反应
             + 汇率或金融稳定的附加反应
```

实际调整应具有平滑性，避免政策利率每月剧烈跳动。

### 11.2 传导渠道

- 银行贷款与存款利率
- 企业投资与居民耐用品消费
- 汇率和资本流动
- 股票与债券估值
- 房地产和抵押品价值（后期）
- 通胀与政策可信度预期

零利率、流动性陷阱、资本外逃和银行惜贷应可能出现，而不是保证每次降息都能刺激经济。

## 12. 证券市场

### 12.1 股票

- 企业可首次发行或增发股票融资。
- 股票价格由订单簿交易形成，不由“公司净资产 × 固定倍数”直接决定。
- 投资者依据利润、分红、增长预期、利率、风险和市场情绪提交订单。
- 企业可以分红、回购、增发、私有化或破产清算。

### 12.2 债券

- 政府和企业发行不同期限与票息的债券。
- 债券价格受政策利率、期限、通胀预期和违约风险影响。
- 新发行市场为融资服务，二级市场提供流动性和市场利率信号。

### 12.3 市场波动

证券市场允许偏离基本面，但偏离应来自杠杆、流动性、趋势交易和预期变化，而不是无来源的随机数。

## 13. 保险系统

保险产品初版提案：

- 企业财产险
- 营业中断险
- 货运险
- 信用保险
- 基础人身保险（居民聚合）

保费由预期损失、风险附加、运营成本和资本成本决定。低估风险会提高短期利润，但在重大事件中可能造成偿付能力危机。

## 14. 财政政策

财政政策通过以下路径影响经济：

- 政府采购直接形成商品需求。
- 公共投资提高未来基础设施与生产率。
- 转移支付改变居民可支配收入。
- 税收改变消费、投资和企业现金流。
- 国债发行影响利率、金融机构资产配置与财政可持续性。
- 补贴和产业政策改变行业利润与资源配置。

财政乘数不是固定参数，而应受到失业率、进口倾向、货币政策、居民储蓄倾向和产能约束影响。

## 15. 国际贸易与汇率

### 15.1 贸易

进口需求受国内需求、国内外相对价格、汇率、关税和供应可靠性影响。

出口需求受外国收入、产品竞争力、汇率和贸易政策影响。

### 15.2 汇率制度

国家可采用：

- 浮动汇率
- 管理浮动
- 盯住汇率
- 资本管制下的行政汇率

汇率变化影响进口成本、出口竞争力、外币债务、通胀和资本流动。

## 16. 公司与国家的双向影响

### 16.1 国家影响企业

国家影响企业：

- 市场需求与收入分布
- 税率、补贴与政府采购
- 工资、技能和劳动法规
- 利率、信贷和资本市场条件
- 汇率、关税和资本管制
- 基础设施、能源和物流成本
- 行业准入、国有竞争者和价格管制

### 16.2 企业影响国家

企业影响国家：

- 产出、投资、就业和工资
- 税收和外汇收入
- 市场集中度与定价能力
- 技术进步与生产率
- 银行风险和证券市场稳定
- 进口依赖与出口竞争力
- 游说、腐败或公共声誉（后期）

玩家公司早期只是宏观经济中的小主体；当市场份额、就业或金融负债达到一定规模后，其决策才会显著影响国家。

### 16.3 局部经济地理（已确认）

地图地块属于某个地区、城市或通勤圈。企业的地理位置会决定其最先影响哪些居民群体和本地企业。

企业新增岗位的初步传导顺序为：

1. 企业发布具有技能、工资和数量要求的岗位。
2. 优先与通勤范围内符合技能的失业者或在职求职者匹配。
3. 招聘困难会推高对应技能工资，扩大培训、跨区通勤或迁入需求。
4. 获得工作的居民提高劳动收入和消费能力，并可能从低收入或失业群体流向更高收入群体。
5. 新增本地消费提高周边零售与服务需求，形成间接就业。
6. 企业采购提高本地供应商收入，也可能把需求泄漏给外地或进口供应商。
7. 人口和商业活动迁入会扩大税基，但也可能推高房租、土地价格和拥堵。

公司对地区的影响至少包括：

- 直接就业、工资和员工技能积累
- 本地供应链订单与间接就业
- 员工消费形成的诱发需求
- 人口迁入、通勤流量和住房需求
- 地方税收、基础设施负担和公共服务需求
- 土地价格、房租、污染、拥堵和生活成本
- 对单一大型雇主的依赖风险

企业关闭、裁员或迁出时，上述链条可以反向运行。大型企业可能使周边地区繁荣，也可能因房租上涨、低技能居民被挤出或环境成本而扩大不平等。

局部影响通过通勤时间、交通可达性、技能匹配和供应链距离逐渐衰减，不采用固定圆形范围内统一加成。

## 17. 每月结算顺序提案

为避免同一周期内的因果循环产生不稳定结果，每月按固定顺序结算：

1. 更新预期、政策和外部冲击。
2. 企业形成销售、生产、雇佣、投资和融资计划。
3. 居民形成劳动供给、消费和储蓄计划。
4. 银行进行授信和贷款定价。
5. 劳动力市场匹配并支付工资。
6. 企业采购投入品并组织生产。
7. 商品市场产生订单、交货、缺货和库存变化。
8. 支付利息、租金、税费、保费和转移支付。
9. 处理违约、赔付、破产和银行损失。
10. 更新价格、工资、汇率和资产价格。
11. 关闭账期，核对资产负债与国民经济账户。
12. 更新预期、信用评级和下一期计划。

每日证券交易和零售订单可在月内累计，月底进行正式财务结算。

## 18. 冲击与不确定性

随机性只用于表达无法预测的事件，不替代经济机制。

可出现的冲击包括：

- 农业歉收、能源中断和自然灾害
- 技术突破和产品质量事故
- 银行挤兑与金融恐慌
- 政策变动、贸易制裁和战争风险
- 居民信心、投资信心和风险偏好变化

所有随机事件应使用可复现的种子，并向玩家提供可观察线索、概率范围或事后解释。

## 19. 玩家可观察信息

不是所有数据都应绝对准确。不同国家的数据透明度会影响：

- 统计发布频率与修订幅度
- 企业财报真实性
- 银行坏账暴露程度
- 政策意图可预测性

玩家至少应能观察：

- GDP、失业、通胀、利率和汇率
- 居民收入、消费与财富分布
- 各商品价格、销量、库存和缺货率
- 行业产能、利润、债务和集中度
- 银行信贷、坏账、资本和流动性
- 政府预算、债务和政策规则
- 贸易差额、外汇储备和外债

## 20. 数值稳定与可验证性

- 核心账户每个结算期必须通过借贷平衡检查。
- GDP 三种核算口径必须进行差异检查。
- 价格和工资以渐进方式调整，避免单周期无限振荡。
- 企业扩产、招聘、贷款和消费都有时间延迟。
- 经济变量可以极端恶化，但不能出现负库存、负人口或无来源存款。
- 游戏开发模式应提供经济审计面板，显示资金来源与去向、部门余额和异常账户。

## 21. 初版实现边界提案

为了先形成严谨闭环，第一阶段建议只做：

- 1 个完整运行的国家，另外 2 至 3 个国家使用较简化的外国部门模型。
- 约 12 至 20 个居民原型，并按地区形成动态聚合群体单元。
- 20 至 30 类商品。
- 玩家公司 + 关键大型企业 + 行业竞争者聚合。
- 央行、财政部、3 至 5 家商业银行、1 至 3 家保险公司。
- 商品市场、劳动力市场、贷款市场、股票和债券市场。
- 固定汇率/浮动汇率、贸易和基本资本流动。

等单国闭环通过会计和稳定性验证后，再让所有国家以相同细度同时运行。

## 22. 当前建议采用的关键决策

以下是设计者建议及当前确认状态：

1. 采用**存量-流量一致（SFC）+ 聚合主体的 Agent-Based Model**，而不是单纯宏观公式。
2. **[已确认]** 商品市场采用企业分散报价、买方选择、库存与订单反馈，而不是每期瞬时均衡或统一强制价格。
3. 生产采用**固定投入系数 + 产能 + 劳动 + 生产率**，优先表现供应链瓶颈。
4. **[已确认]** 居民采用约 12 至 20 个动态原型并按地区聚合，不逐户模拟；就业、财富和迁移会推动群体间人口流动。
5. 国家制度采用**政策参数组合**，不使用简单制度标签直接给予加成。
6. **[已确认]** 先做一个完整国家闭环，其他国家暂时简化；验证后再扩展多国完整模拟。
7. **[已确认]** 初版采用有限数量的核心商品，不从 50 类以上开始；商品使用分层目录，可在保持会计兼容的情况下逐步细分。
8. 核心经济过程尽量确定性；随机冲击可复现且有来源。
9. **[已确认]** 商品、服务和金融工具采用独立系统：商品是可交付产品或许可，服务是履约活动，证券等属于金融权利凭证。
10. **[已确认]** 商品分类树与生产依赖图相互独立：分类树表达归属，生产配方图表达投入、加工和产出关系。
11. **[已确认]** 生产配方采用“关键投入 + 类别投入 + 有限替代组”：关键投入不可替代，替代品必须技术可行并具有独立效率与质量影响。
12. **[已确认]** 玩家公司通过岗位匹配、工资、供应链、消费、迁移、房租和地方财政反向影响周边居民群体及全国经济。
13. **[已确认]** 消费者按地区居民群体在多个品牌产品间分配份额；真实质量、质量认知、品牌口碑和售后履约分别模拟。

## 23. 下一轮需要共同决定的问题

1. **[已确认] 模拟范围**：首个版本只完整模拟玩家所在国，其他国家先作为简化外国部门；单国闭环通过会计与稳定性验证后，再扩展为多国完整模拟。
2. **[已确认] 商品颗粒度**：初版采用有限数量的核心可交付商品，不从 50 类以上开始；通过统计节点、可交易叶子和折叠节点支持后续细分，并使用独立生产依赖图描述产业链。
3. **[已确认] 居民颗粒度**：采用约 12 至 20 个动态居民原型并按地区聚合，不逐户模拟；玩家公司可改变附近就业、收入、迁移、消费和财富流动。
4. **[已确认] 市场风格**：商品市场采用企业分散报价、消费者选择、库存和积压订单反馈；消费者根据价格负担、质量认知、品牌口碑、售后、可得性和偏好形成品牌份额。
5. **计划经济表现**：是否允许行政定价、配给、计划订单、国有银行信贷指导和软预算约束同时存在？
6. **玩家信息**：玩家应能看到完整真实数据，还是数据透明度由国家制度、审计和情报能力决定？
7. **失败强度**：企业和银行是否允许真正破产清算，还是玩家公司可通过重组避免直接结束游戏？
8. **游戏节奏**：现实一年大约对应多少分钟实际游玩时间？
